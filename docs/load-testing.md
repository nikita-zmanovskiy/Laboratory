Нагрузочное тестирование, чтобы проверить:

- устойчивость backend под параллельными запросами
- время ответа API
- корректность rate limit
- стабильность генерации
- работу логирования
- отсутствие падений при одновременной активности нескольких пользователей

Что тестируется

Основные сценарии:

1. Создание класса преподавателем.
2. Подключение ученика по коду класса.
3. Генерация текстового ответа.
4. Генерация изображения.
5. Получение логов класса.
6. WebSocket-уведомления о новых событиях.

Подготовка окружения

Перед запуском тестов нужно поднять backend и зависимости:

cd backend
npm install
npm run build
npm run test:all
